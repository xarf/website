## Community Guidelines

We are committed to providing a welcoming and inclusive environment.

### Code of Conduct

All community members must:

- **Be respectful** - Treat everyone with respect and consideration
- **Be constructive** - Provide helpful, actionable feedback
- **Be inclusive** - Welcome diverse perspectives and experiences
- **Be patient** - Help newcomers learn and grow
- **Be professional** - Keep discussions focused and productive

**[Read Full Code of Conduct ‚Üí](https://github.com/xarf/xarf-spec/blob/main/CODE_OF_CONDUCT.md)**

### Getting Help

- **Documentation**: Start with our [docs]({{ site.baseurl }}/docs/introduction/)
- **Search First**: Check if your question was already answered
- **Be Specific**: Provide context, examples, and error messages
- **Follow Up**: Let others know if solutions worked

## Contributing

We welcome contributions from everyone! Here's how to get started:

### Quick Contributions

No coding required:

- **Report bugs** you encounter
- **Suggest improvements** to documentation
- **Share use cases** and implementations
- **Answer questions** in discussions
- **Spread the word** about XARF

### Code Contributions

Help build the ecosystem:

- **Libraries** - Implement XARF in new languages
- **Tools** - Create utilities and integrations
- **Documentation** - Improve guides and examples
- **Specification** - Propose new event types

**[Read Contributing Guide ‚Üí]({{ site.baseurl }}/community/contributing/)**

## Governance

XARF is developed as an open standard with community input.

### Decision Making

- **Specification changes** require community discussion and consensus
- **Major versions** announced 12 months in advance
- **Breaking changes** require exceptional justification
- **Community feedback** drives the roadmap

### Maintainers

Current specification maintainers:
